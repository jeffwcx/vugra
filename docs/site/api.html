API

Stable public surface

Application code should depend on `pkg/vugra`, `pkg/signal`, and `pkg/system`. Internal packages are implementation details for compiler, runtime, layout, renderers, and tooling.

pkg/vugra

`pkg/vugra` is the public runtime facade and type alias layer. It exposes component IR types, runtime mount options, renderer interfaces, layout constraints, and host-neutral system types.

SymbolKindDescription
Mount(component, state, target, options)FunctionMounts a Vugra component into a renderer and returns an `*App`.
NewSoftware(width, height)FunctionCreates a software renderer for PNG/test output.
Emit(event)FunctionMarker used by generated/component logic for component event semantics.
AppType aliasRuntime app handle. Important methods include `Flush`, `Unmount`, `Resize`, `DispatchPointerEvent`, `DispatchScroll`, `DispatchKey`, `DispatchTextInput`, input selection APIs, and document text selection APIs.
TextSelectionType aliasRenderer-neutral text selection with Vugra input `ID` or document text scope, binding name when relevant, and rune-offset `Start`/`End` range. It deliberately avoids browser DOM `Range` objects so native and wasm hosts share the same API.
TextSelection, SetTextSelection, SelectedTextMethodsQuery, set, collapse, and read focused or targeted text input selections.
DocumentTextSelection, SelectedDocumentText, ClearDocumentTextSelectionMethodsQuery, read, and clear renderer-owned selection over non-control rendered document text such as documentation headings and paragraphs.
StateType aliasRuntime state maps for signals, methods, and event methods.
Component, Node, Element, TextType aliasesFramework-owned component IR.
Interpolation, Conditional, RepeaterType aliasesTemplate binding IR nodes.
ComponentInstance, DynamicComponentType aliasesComponent composition IR.
Prop, PropDef, ProvideDef, InjectDef, Slot, EmitDef, Lifecycle, EventHandlerType aliasesComponent metadata used by generated code and runtime.
Constraints, Measurer, FixedMeasurerType aliasesLayout inputs and text measurement contracts.
LayoutEngine, LayoutEngineGo, LayoutEngineCSSType alias and constantsRuntime layout engine selection exposed for generated/native/wasm entry points.
Renderer, Command, Rect, SoftwareRendererType aliasesRenderer-neutral output contract.

Mount Example

target := vugra.NewSoftware(800, 600)
app := vugra.Mount(component.Component(), component.NewRuntimeState(nil), target, vugra.Options{
    Styles:      component.Styles(),
    Constraints: vugra.Constraints{Width: 800, Height: 600},
    Measurer:    vugra.FixedMeasurer{CharWidth: 8, LineHeight: 20},
})
app.Flush()

pkg/signal

`pkg/signal` provides explicit reactive values for Go component state.

SymbolDescription
signal.Signal[T]Generic signal with `Get`, `Set`, `Update`, `Subscribe`, `GetAny`, and `SetAny`.
signal.New[T](initial)Constructs a generic signal.
signal.Int, signal.Bool, signal.StringConcrete signal types used by examples and template metadata.
signal.NewInt, signal.NewBool, signal.NewStringConstruct concrete signals with initial values.

pkg/system

`pkg/system` is the host boundary for capabilities that should stay replaceable across native, wasm, tests, and sandboxed runtimes.

SymbolDescription
FileSystemInterface for `ReadDir`, `Stat`, `Mkdir`, `Rename`, `Remove`, and `Duplicate`.
EntryRenderer-neutral file metadata with name, path, kind, size, and modified time.
OSFileSystemNative `os` backed implementation.
WebFileSystemWasm browser implementation backed by sandboxed Web Storage, with an in-memory fallback for non-browser wasm hosts.
MockFileSystemIn-memory implementation for examples and tests.
SetFileSystem, DefaultFileSystemManage the process-wide file system backend used by package helpers.
GUIInterface for reading and setting host window chrome.
WindowChrome, WindowControls, WindowTitlebarModeNative window chrome model that can expose layout tokens to Vugra.
WindowControlsXToken and related constantsToken names consumed by layout through `env(vugra-window-controls-*, fallback)`.

Wasm API

The wasm API is compiled only for `GOOS=js GOARCH=wasm` and bridges browser events and canvas rendering into Vugra runtime semantics.

SymbolDescription
NewCanvasRenderer(canvas, width, height)Creates a canvas 2D renderer for Vugra render commands. The renderer uses CSS-pixel logical coordinates, a device-pixel-ratio backing store, and the current SVG subset for `rect`, `circle`, and simple path commands.
NewCanvasMeasurer(canvas)Creates a wasm text measurer backed by the canvas 2D `measureText()` API, so browser layout measurement and canvas text rendering use the same host font metrics path.
InstallPointerEventsInstalls a click-oriented pointer bridge.
InstallPointerEventDetailsInstalls click, hover, drag, double click, and context menu event bridging.
InstallScrollEventsInstalls wheel event bridging.
InstallKeyboardEventsInstalls key dispatch for focus/global keyboard handling, including Tab and Shift+Tab focus traversal, Backspace, Delete, ArrowLeft, ArrowRight, Home, End, and Mod+A editing for focused text inputs through the runtime key path.
InstallTextEventsInstalls text input dispatch for printable keys, IME composition commits, and plain text paste events at the focused runtime caret or selection.
SyncAccessibility(containerID, commands, focusedID)Mirrors the current renderer command frame into a hidden DOM accessibility tree with roles, names, checked state, disabled state, focused state, and layout metadata.
InstallAccessibilityEvents(containerID, dispatch)Installs the legacy hidden DOM accessibility click bridge that maps mirror activation back to Vugra pointer dispatch coordinates.
InstallAccessibilityEventHandlers(containerID, handlers)Installs hidden DOM accessibility bridges for focus, click, key, and text actions.
SetStatus(id, text)Updates a host DOM status element.

API stability note: the facade packages are the intended app-facing surface. `internal/*` packages can change as the compiler/runtime boundary matures.

This page documents the current public API shape in the repository.

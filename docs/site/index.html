Documentation Site

Build Vugra components with standard HTML templates and Go logic.

Vugra compiles `.vue` single-file components into framework-owned IR, mounts them through a Signal-driven runtime, and renders through backend-neutral commands for native and wasm hosts.

Current Status

Available now SFC parsing, HTML template parsing, Go metadata extraction, IR validation, Signal runtime updates, component imports, flex/grid/block layout, event routing, native window support, software rendering, Vello native rendering, and wasm canvas bundles.

Not complete yet WebGPU wasm rendering, full browser DOM behavior, full CSS cascade, richer widgets, platform accessibility bridges, and production LSP integration remain future work.

Quick Start

go test ./...
go run ./cmd/vugra check examples/counter/Counter.vue
go run ./cmd/vugra wasm examples/counter/Counter.vue /tmp/vugra-counter-wasm
	go run ./cmd/vugra wasm-serve /tmp/vugra-counter-wasm

Open `http://localhost:8000` after starting the server. The HTTP server is only a static file host for the generated wasm assets.

Design Boundary

Vugra uses HTML syntax, but it is not a wrapper around browser DOM semantics. Components compile into Vugra IR first, then runtime/layout/render layers decide how to present the same component on native or wasm targets.

The canonical docs site is implemented in `docs/site/App.vue` and built with `vugra wasm docs/site`; these static pages remain as compatibility reference material.
